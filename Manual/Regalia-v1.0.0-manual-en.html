Chapter 1
App Introduction

Regalia v1.0.0 is a powerful Android chess playing and analysis app. This version includes the world-class open-source engine Stockfish 18 (arm64-v8a-dotprod native build), providing professional-grade playing, review, and training experience for everyone from beginners to advanced players.

18 Stockfish Engine Version
6+SL AI Difficulty Levels
8 MultiPV Lines

Core Features

  • Built-in Stockfish 18 engine, arm64-v8a-dotprod native build, automatically utilizes NEON accelerated instruction set
  • 6 preset AI difficulty levels + SL tier (custom Skill Level) + ⚙️ manual settings, suitable for players of all levels
  • AI thinking progress bar displays search depth, node count, speed, and evaluation score in real-time
  • Ponder mode — AI continues thinking during opponent's turn, recommended to enable while charging for faster subsequent moves
  • MultiPV multi-line analysis (1-8 lines), displaying multiple engine-recommended variations simultaneously
  • Variation system: engine predicted continuations and alternative variations displayed in move record and review (💬 variation toggle)
  • Full review mode: step-by-step navigation, evaluation trend chart, move classification and key move annotation, analyze all
  • ECO opening library with search, opening recommendations, and family filter
  • Syzygy online endgame tablebase query (win/draw/loss determination for 7-piece and below endgames)
  • WDL win/draw/loss probability display (enabled by default), intuitively understand position advantage
  • Evaluation score includes SF18 engine badge, search depth D indicator, node and speed information
  • Square control range heatmap (🌈/🌗 toggle), with arrow indicators and square control info panel
  • AI hint multi-line display, showing all recommended moves and scores
  • Chess principles panel, providing real-time top 10 chess principle reminders
  • Engine configuration export/import (key=value text format), convenient for multi-device sync
  • FEN copy/import, supporting quick restoration of any position
  • Setup mode, freely edit board positions with undo/redo/reset/clear
  • Compatible with Android 5.0 through Android 15, adapted for Xiaomi HyperOS 3

Implementation Details: Module Architecture

The app adopts a multi-module layered architecture. Source code is located in the assets/chess.src/ directory and merged into a single chess.html file via the build script build-chess.sh, loaded and run by Android WebView:

First Launch

On first launch, the app displays a loading screen: ♔ King Icon ( #E8E8F0) + app name + Progress Bar ( #d4a017→#ffd700 gradient). The engine binary file is automatically extracted to the local private directory and granted execute permission; after the progress bar reaches 100%, "Engine Ready!" is displayed, and the loading screen fades out in 0.5 seconds.

Implementation Details: Engine Initialization Flow

StockfishNative.java copies the engine binary from assets to the app's private directory (context.getFilesDir()) in onCreate(), and grants execute permission via JNI call to nativeChmod(). After WebView loads chess.html, the JS layer calls AndroidBridge.startEngine() to start the UCI process; the engine returns uciok via stdout, then _engineReady=true is set.

ai-bridge.js — Engine Ready Callback function onEngineReady(){ _engineReady=true; _needNewGameForEngine=true; // Need to re-issue ucinewgame after crash recovery _updateLoadingStatus('Engine Ready！',100); // 0.5s Fade OutLoading Screen setTimeout(function(){_hideLoadingOverlay()},500); }

Foreground Service Notification

When the engine process is running, the app displays a Foreground Service Notification, continuously showing engine process information in the System Notification Bar. This notification informs the Android system that the engine process is performing important computations, preventing the system from terminating the engine process due to memory reclamation or power-saving policies, ensuring that gameplay and analysis are not interrupted.

Bilingual Notification：The text content of the Foreground Service Notification is displayed bilingually, following the app's current language setting. In Chinese mode, the notification title and content are in Chinese; in English mode, they are in English, consistent with the interface language.
Notification Permission：On Android 13 (API 33) and above, the app proactively requests Notification Permission (POST_NOTIFICATIONS) when the engine is first launched, to ensure the Foreground Service Notification can be displayed normally. If the user denies permission, the engine still runs normally, but the System Notification Bar will not display engine status information, slightly increasing the risk of the engine process being terminated by the system.

Control Heatmap

Click the 🌈 button on the toolbar to enable Control Range display; board squares will be dynamically colored based on both sides' Control Strength. The heatmap uses HSL color space, encoding control information through three dimensions: Hue, Saturation, and Lightness.

Heatmap Coloring Algorithm

Heatmap colors are determined by three parameters:

  • Advantage Direction（adv）：Friendly control count − Opponent control count ÷ Total control count. adv > 0 is friendly advantage, adv < 0 is opponent advantage. Determines Hue
  • Control Strength（str）：Total control count ÷ 8 (capped at 1.0). Determines Saturation and Lightness
  • No Control：When neither side has control, a fixed color is used #3a2020

Hueadv≥0: 280 − adv×60
adv<0: 280 − adv×80		≈220°~340°Purple-blue=Equal, blue=friendly advantage, red=opponent advantage
Saturation0.50 + str×0.4050%~90%More control = more vivid
Lightness0.48 − str×0.1236%~48%More control = darker

Implementation Details: Control Heatmap

game-logic.js getCtrlMap() iterates over all board squares, calling attacked() to obtain attack targets, aggregating them into an 8×8 control matrix. Each element records {white: [{piece, position}], black: [{piece, position}]}。

ui.js _getSqBg() calculates HSL colors based on the control matrix to overlay board square backgrounds; _updateArrows() draws control arrows in the SVG overlay layer, using persistent SVG elements to avoid repeatedbuild DOM。

If the opponent's actual move matches the engine's prediction (Ponder Hit), the engine can immediately utilize the already analyzedresult，with near-zero latencybacknext stepmove；if inconsistent（Ponder Miss），enginestopbackgroundsearchand re-analysisnewposition。

Ponder (background thinking) is a feature defined in the UCI (Universal Chess Interface) protocol: after the engine completes its move, it can continue in the background to predict the opponent's reply and pre-analyze that predicted position. If the opponent's actual move matches the engine's prediction (Ponder Hit), the engine can immediately use the already-analyzed result and return the next move with near-zero latency; if not (Ponder Miss), the engine stops the background search and re-analyzes the new position.
